In a previous version a ScientificProperty was distinguished from a NumericalProperty.  In this version both of those classes have been removed and the (no longer abstract) class ComponentProperty is used as a consolidation of them.  Note, that the container classes NumericalProperties and ScientificProperties still exist as features of a ComponentModel, but they now both contain ComponentProperties.  It is still not obvious when one would mark a property as "scientific" rather than "numeric," though it is expected that some hueristics will come out of the CMIP5 Questionairre Excercise (If not, then these container classes can be removed in a future version).
