The difference between a boundary condition numerical requirement and a physical modification conformance is in the eye of the designer: it is possible for a particular model component to see a physical modification change in another component as a boundary condition change in that component. <br /></p><p>For example, if we had a set of experiments which were aimed at addressing the effect of fresh water inflow in the north atlantic, one model might well represent that as a boundary condition on the north atlantic ocean, another as a change in a fresh water parameterisation. These would be described in the conformance class, but from a model experiment designer perspective either physical modification or boundary condition would be valid.<br /></p>
