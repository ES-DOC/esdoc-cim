Overview - DRAFT

This document describes the file format of CIM2 specialisations for describing CMIP6 models.

Files

A seperate file is created for each realm, and, for each realm, its key properties, its grid, each of its processes. All files contain three different type of variables:
  1. Metadata variables which state which model component is being described, who contributed to creating the file contents and the quality control status of the file contents
  2. Internal variables required by the ES-DOC tools which use the files
  3. The variables which describe a model conponent

Metadata variables

A file must contain the following metadata variables:
CONTACT
The point of contact for any questions about the file. There are no restrictions on the value (e.g. CONTACT ='David Hassell, NCAS, email@address').
AUTHORS
The names of all those contributed to the file contents. There are no restrictions on the value (e.g. AUTHORS = 'David Hassell, NCAS, email@address; Charlotte Pascoe, NCAS, email@address').
QC_STATUS
The quality control status of the file ccontent. Allowed values are 'draft' (content still being developed), 'complete' (content complete and ready to be ingested into ES-DOC), 'accepted' (accepted and ingested by ES-DOC - this value should only be set by ES-DOC developers). Examples: QC_STATUS = 'complete'.
ID
A unique identifier for the file of the form 'cmip6.', or 'cmip6..' where represents the model realm being described (e.g. atmosphere) and represents the realm component being described, one of key_properties, grid, or a model process (e.g. cloud_scheme).
These variables are preset in model component file templates and it is expected that they will be changed during the creation of a new file.

Internal variables

A file must contain the following internal variables, which are preset in the model component file templates and should be not changed during the creation of a new file:
_TYPE
The CIM construct type of the model component (e.g. _TYPE = 'cim.2.science.process'
